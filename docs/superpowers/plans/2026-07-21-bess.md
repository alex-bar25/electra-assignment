# Minimal BESS Implementation Plan

> **For agentic workers:** Use the approved inline execution workflow, TDD, and one commit/review checkpoint per task.

**Goal:** Add optional deterministic BESS boost, spare-grid charging, SoC ticking, OPS visibility, and runnable examples without changing grid-only behavior.

**Architecture:** The service owns mutable BESS state and dispatch. The allocator remains pure and receives an explicit station-supply limit. Time advances only through an explicit HTTP tick under the existing mutex.

**Tech stack:** Go 1.26 standard library, existing Python scenario runner, Docker Compose, and Postman Collection v2.1.

## Constraints

- No background goroutine or wall-clock battery simulation.
- Positive BESS power is discharge; negative is charge.
- EV allocations are never reduced to charge BESS.
- SoC remains between configured minimum and `100%`.
- Grid import remains between zero and grid capacity.
- Existing behavior is unchanged when BESS is omitted.
- No new dependencies.
